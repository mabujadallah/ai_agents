/ LLM

ما تعرفه فعلًا — ChatGPT، Claude، Gemini.

// large language model

LLM هو نموذجُ ذكاءٍ اصطناعيٍّ مُدرَّبٌ على نصوصٍ ضخمة، يَتنبَّأ بالكلمة التَّالية بناءً على ما سَبَق.

المنظور الأبسط: function تأخذ نصًّا، وتُرجع نصًّا.

PROMPT ◀
«كيف أُظهِر رسالة للمستخدم في Android?»
f(x)
LLM
stateless · text → text
OUTPUT ▶
Toast.makeText(this,
"مرحباً",
Toast.LENGTH_SHORT).show()

بلا ذاكرة، بلا أدوات، بلا مُبادرة. التَّطبيقات الشَّهيرة هي واجهاتٌ فوق LLMs.

ChatGPT
Claude
Gemini
Copilot
// same tool, two outcomes

ما يُجيده LLM… وما يَعجِز عنه.

✓ WORKS

prompt → LLM → كود جاهز

سؤال عام موجود في ملايين المرجع — LLM يجيب فوراً.

// more prompts that just work
«كيف أُظهِر قائمة بالعناصر في Android?».
«ما الفرق بين Activity و Fragment?».
«كيف أحفظ بيانات المستخدم محلياً في التطبيق?».
✗ FAILS
«لماذا الزر لا يعمل في تطبيقي?»
prompt → LLM → I don't know

LLM لا يرى كودك ولا ملفات مشروعك — ما عنده إلا ما كتبته في السؤال.

// more prompts that hit a wall
«لماذا تطبيقي يتوقف فجأة?».
«لماذا البيانات لا تُحفظ بعد إغلاق التطبيق?».
«لماذا الصورة لا تظهر في شاشتي?».

// how agents think

كيف يُفكِّر الـ Agent؟

ReAct · Reason → Act → Observe
R
فكِّر — Reason
ما الخطوة التَّالية؟ أيُّ أداةٍ أستخدم؟
A
نفِّذ — Act
استدعِ الأداة — ابحَث، اقرأ، أرسِل.
O
راقِب — Observe
اقرأ النَّتيجة ← انتهيت أم أُكرِّر؟
مثال · «احجُز أرخص طيران لدبي غدًا»
── cycle 01 ──
R سأبحث في المواقع.
A search_flights("Dubai")
O 4 رحلات، أرخصها 249$.
── cycle 02 ──
R أتحقَّق من تفاصيل الأرخص.
A get_details("EK204")
O مباشرة، 9ص، مقاعد متاحة.
── cycle 03 ──
R الرحلة مناسبة، سأحجز.
A book_flight("EK204")
O ✓ تم الحجز — PNR1234.
// self-correcting loop

الـ Agent يُصحِّح نفسه.

بلا تدخُّلٍ منك — يُكرِّر حتَّى يُتقِن.

المحاولة الأولى
// Writer يكتب v1
fun sum(list: List<Int>): Int {
  return list.sum()
}

←

// Writer يُصلح v2
fun sum(list: List<Int>): Int {
  if (list.isEmpty())
    return 0
  return list.sum()
}
✓
Critic: جميع الاختبارات تَنجح — مُجاز.
←
✓
انتهى
بدون تدخُّل
بشريّ
💡

/ Prompt Engineering

كيف تتحدَّث مع النَّموذج.

// what is a prompt?
Prompt = النَّصُّ الذي تُرسله للنَّموذج. تَعليمات، سياق، أمثلة، أو شكلٌ مَطلوبٌ للإخراج — كلُّها تَدخل ضمن الـprompt. جودة الإجابة تَبدأ من جودة الـprompt.
// don't — أربعة أنماط تُفسد النتيجة

ماذا لا تفعل؟

01 · غامض VAGUE
لا تعرف ماذا تريد
💬 «اكتب لي شيئًا عن الذكاء الاصطناعي»
← لا جمهور، لا طول، لا نبرة. ستحصل على إجابة عامة لا تفيدك.
02 · بلا سياق NO CONTEXT
تفترض أنه يقرأ ذهنك
💬 «أصلِح هذا الكود» — بدون أن ترسل الكود
← يخمّن ماذا تقصد. يجيب على شيء مختلف تماماً.
03 · أسئلة كومة COMPOUND
3 أسئلة في رسالة واحدة
💬 «اشرح Kotlin، وقارن Fragments وActivities، واكتب ViewModel»
← ستحصل على 3 إجابات سطحية بدل إجابة واحدة عميقة.
04 · بلا شكل NO FORMAT
لا تحدد شكل الإجابة
💬 «لخّص لي هذا المقال»
← قد يأتي كفقرة، نقاط، أو جدول. لا يمكنك البناء عليه برمجياً.
// do — أربعة قواعد ترفع الجودة

ماذا تفعل؟

01 · كن محدداً BE SPECIFIC
حدّد الجمهور والنبرة والطول
💬 «اكتب 3 جُمل عن AI لطالب جامعي، نبرة حماسية»
← جمهور + نبرة + طول = مساحة جواب أضيق وأدق.
02 · أعطِ السياق GIVE CONTEXT
أرسل كل ما يحتاج معرفته
💬 «هذا كود Kotlin فيه NullPointerException بسطر 12: [الكود]. أصلحه واشرح لماذا حدث.»
← كل ما يعرفه = كل ما كتبته له. لا تخمين.
03 · أعطِه دوراً SET A ROLE
«أنت X» ينقله لمستوى أعلى
💬 «أنت مطوّر Android متمرّس. راجع هذا الكود وحدّد المشاكل التقنية.»
← «أنت X» يُفعّل سلوك المحترف في X تلقائياً.
04 · حدّد الشكل SPECIFY FORMAT
قل له كيف تريد الجواب
💬 «أعطني الإجابة كـ JSON بالحقول: issue, line, fix»
← شكل محدد = إخراج يمكن استخدامه مباشرة في الكود.
// same task · same model · different prompt

// FULL PROMPT · annotated
① ROLE + CONSTRAINTS
أنت مطوِّر Android متمرِّس
متخصِّص في Kotlin وJetpack Compose.
— لا تقترح حلولًا خارج بيئة Android.
— أجب بالعربية، الكود بالإنجليزية.
— الردود: مشكلة → سبب → حل → كود.
② CONTEXT
المشروع: تطبيق Android يعرض قائمة منتجات.
المشكلة: التطبيق يتجمَّد عند فتح الشاشة
الرئيسية (ANR بعد 5 ثوانٍ).
// MainActivity.kt — 30 سطرًا مُرفقة
③ FORMAT
الشكل: JSON array
[ { line, issue,
severity: critical|warning|nit,
fix, code_example } ]
① ROLE + CONSTRAINTS
تحديد الشَّخصيَّة والحدود.
«أنت X» يَنقل النَّموذج لمنطقة أداءٍ أعلى. القيود تَقطع التشتُّت قبل أن يبدأ.

/ Vibe Coding

برمجة بالكلام — لا بالكود.

// coined by andrej karpathy · 2025
صِف ما تريد بلغتك الطبيعية، النموذج يكتب الكود، أنت تُجرِّب وتُكرِّر. لا تحتاج أن تفهم كل سطر — تحتاج أن تعرف ماذا تريد.
// andrej karpathy · feb 2025
